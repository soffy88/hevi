## 1. L1 Concept（锁定）

**一句话**：他赢了战争之后，真正的难题才开始——这一集讲清楚郡县制凭什么赢过分封制。

**三段叙事逻辑**：刺杀（统一是有人拿命阻止的事，公元前 227）→ 统一（六年后他坐在同一座大殿里，公元前 221）→ 治理（分封还是郡县：一场《史记》全文记录的真实辩论）。

**时间结构**：冷开场在前 227 年，正片主体在前 221 年，同一座咸阳宫大殿，两套 dressing 承载六年跨度——一个场吃整集，符合"一主场"纪律。

**本期回答的"教材没空回答的为什么"**：郡县制凭什么赢——不是嬴政拍脑袋，而是分封制刚刚给出过 800 年级别的失败样本（李斯原话：属疏远，相攻击如仇雠），且郡县提供了新机制：守/尉/监三权分置、流官任免、俸禄替代封地。

## 6. L6 Generation 技术备注

1. **多人镜**（S01/S02/S13/T01/T03/W 系）：compose→img2img 已验证路径，不训 LoRA。
2. **Subject3D**：6 具名角色全建（约 172s/个 CPU，合计约 17 min，忽略不计）。执行 2D/3D 共存原则：2D 管身份保真，3D 管视点自由——嬴政/荆轲跨 10+ 镜多方位，是 3D 锚的主要受益者。
3. **口型清单**：T02/T05/T07 共 3 镜，全部静态构图正/侧面——落在已验证能力的舒适区。
4. **高风险镜**：S07 已结构化降险（见 §5.1 S07a/b/c）；S05 保留单独精修预算。

## 11. 决策记录

### DR-1 · 生成通道裁决（2026-07-23）

**裁决**：EP0 的 compose→img2img／多人口型／Subject3D 三项能力走 tongjian avatar 通道，不做 V2 移植前置。

**约束（四条，违反任一即裁决失效）**：
1. **唯一 adapter 封装**：所有对 avatar 通道的调用收敛到单一模块（建议落点：`qnlr_gen_adapter`），director-pipeline 侧只见 adapter 接口，严禁直连 tongjian 内部。
2. **只包不改**：EP0 周期内冻结 tongjian 内部改动，防顺手重构上关键路径。
3. **产物入新范式**：身份锚、合成帧、口型片段按 fingerprint + decision_trail 惯例登记。A-QIN 是全季资产，数据资产不绑通道，不锁死在旧通道存储约定里。
4. **移植窗口**：V2 移植 SPEC 起草于 EP0 复盘周（9 月初），需求清单以 EP0 实测调用模式为底本；EP1–3 考古期（生成压力谷值）为切换缓冲，切换完成目标在 EP5（牧野，下一个演绎重镜期）之前。

**理由**：8 月底硬发布窗口约 5 周，不容 L 级移植加新 SPEC 上关键路径；与 Tide v4 教训的区别——那次是新写业务逻辑写错范式，这次是消费已验证通道，债务被 adapter 封顶在单模块内；EP0 同时充当 V2 移植的需求探针，先垂直切片后定结构，符合既有方法论。
