> 2026-07-14**(见文末完成记录)。阶段2(①②生成质量专项)、阶段3(出品形式接入)未开始。
> 原始状态:草案 v0.1,可交 CC 实施

> 定调(产品决策):**AI 当导演,人当监制。** ①立意 ②剧本 由 AI 生成,人在每一级审核锁定后才放行下游。AI 不是执行工具,是坐在导演椅上的那个。

## 0. 病根诊断:主线为什么产出崩坏

实证真实电影/动画工业前期流程,铁律是一句话:**"剧本是后续工作的基石,剧本不确定,原则上不能开展下面的工作"——上游不锁定,下游不许动。**

完整工业顺序:立意/主创定案 → 剧本 → 制作分解(设计清单)→ 美术设计(视觉风格指南)→ 分镜头剧本(导演剧本)→ 拍摄/录音/剪辑。

**主线跳过了"制作分解、美术设计、分镜头剧本"三步,直接从文字冲到生成。** 这就是崩坏的根因,逐一对应:

| 崩坏症状 | 根因 | 缺失的工业步骤 |
|---|---|---|
| 人物/场景乱跳 | 资产没在生成前被锁定 | 缺"制作分解 → 设计清单",场景/人物/衣着/占位无人锁 |
| 只有旁白没对白 | 主线读的是叙述文字,不是分镜里的台词行 | 缺"分镜头剧本",没把每镜头的对白台词切出来 |
| 对白是文言不是白话 | 文言原文被当剧本直接用 | 缺"剧本重写",剧本应是白话重写,原文只是素材 |
| 不够详细 | 每级没有强制的结构化输出 | 缺整条链的逐级锁定 |

**结论:不是修 bug,是给主线补上缺失的整个前期层。**

## 1. 主线导演流水线:五级强制链(上游锁定才开下游)

```
① 立意 Concept ──锁定──▶ ② 剧本 Screenplay ──锁定──▶ ③ 设计清单 Design List
                                                            │ 锁定
                                                            ▼
   ⑤ 生成/配音/装配 ◀──锁定── ④ 分镜头剧本 Shot List ◀──────┘
   (现有 L1)

铁律:每一级是下一级的 readiness 硬前置。上游未锁定(未通过人审核),下游禁止启动。
AI 生成每一级草案 → 人审核/修改 → 锁定 → 放行下游。
```

这条链就是 v3.1 的 readiness 状态机(4.0.2)升级到"作品级":不只是镜头备齐才开机,是**每一级导演决策锁定才放行下一级**。

## 2. 逐级定义(输入 / AI 生成什么 / 人审核什么 / 锁定产物)

### ① 立意 Concept(AI 生成,人拍板)

- **输入**:素材(小说/大纲/文言原文/一句话想法)+ 用户意图(目标观众、时长、风格倾向)
- **AI 生成**:主题、基调、风格基调、目标观众、艺术品质基准(对应工业"主创定案")。承接 v3.1 Producer(意图→约束)+ StylePack 根变量,但**前置为第一道锁**。
- **人审核**:这个立意对不对、基调要不要调
- **锁定产物** `Concept`:`{主题, 基调, 风格, 目标观众, 时长档, 品质基准}` → 成为下游所有决策的约束根
- **锁定前,不许写剧本**

### ② 剧本 Screenplay(AI 生成,白话、详细,人审核)

- **输入**:锁定的 Concept + 素材原文
- **AI 生成**:完整白话剧本,分场。**关键要求:**
  - **白话**:不管素材是文言还是白话,剧本一律白话重写。文言原文是**素材**,剧本是**重写产物**,不是搬运。(tongjian 的文言→白话转换在此级注入,见第 5 节)
  - **详细**:每一场是结构化的,不是一段散文
  - **分场**:每场 = `{场号, 时间, 地点, 出场人物, 事件, 对白脉络}`
- **人审核**:剧情对不对、白话通不通顺、详略是否合适
- **锁定产物** `Screenplay`:分场剧本,每场结构化
- **锁定前,不许做设计分解**

### ③ 设计清单 Design List(AI 从剧本分解,人审核锁定)

这是主线**当前完全缺失、崩坏的直接根因**。对应工业"制作分解 + 美术设计"。

- **输入**:锁定的 Screenplay
- **AI 生成**:扫描剧本,自动分解出三张待锁定清单(接 L2 五类挂载树):
  - **场景清单**:每个场景 `{环境, 光照, 氛围, 是否主场景}` → 建 scene 资产
  - **人物清单**:每个角色 `{外貌, 衣着, 发型, 性格, 是否主角, 声线}` → 建 character 资产 + 挂载 wardrobe/hairstyle/voice
  - **道具清单**:关键道具 `{外观}` → 建 prop 资产
- **人审核**:清单全不全、每个资产的描述对不对(这个角色该穿什么、这个场景什么氛围)
- **锁定产物** `DesignList`:每一项 = 一个**已锁定的资产**(接 5.0.2 挂载树,身份/衣着/场景/道具全部在此锁死)
- **锁定后,人物/场景/衣着不再变** → 根治"乱跳"
- **锁定前,不许分镜**

> **占位(blocking)说明**:"占位"(谁站在哪、走位)属于④分镜级的镜头内信息,不在③。③锁的是"这个角色/场景长什么样",④锁的是"这个镜头里他站哪、什么景别机位"。

### ④ 分镜头剧本 Shot List(AI 生成,人审核锁定)

对应工业"分镜头剧本(导演剧本)"。这是主线**缺失导致"只有旁白没对白"的根因**。

- **输入**:锁定的 Screenplay + DesignList
- **AI 生成**:把每一场切成镜头序列,每镜头带完整导演信息(七段式 schema 扩展):
  - 镜头号 / 景别(远中近特)/ 机位·摄法 / **画面内容**
  - **【台词行:谁说 + 说什么】** ← 这是对白!必须显式切出"角色A:台词"这样的行,而不是笼统的旁白叙述
  - **占位/走位**:本镜头哪些角色、各自站位/朝向(接机位驱动渲染)
  - 引用哪些锁定资产(哪个角色 + 哪套衣着 + 哪个场景 + 哪些道具)
  - 音乐 / 音响 / 镜头时长
- **人审核**:分镜合不合理、台词对不对、每镜头调用的资产对不对
- **锁定产物** `ShotList`:每镜头是一个 ready 的生成单元(引用已锁资产 + 带台词 + 带占位机位)
- **锁定前,不许生成**

### ⑤ 生成 / 配音 / 装配(现有 L1,基本不动)

- **输入**:锁定的 ShotList
- 逐镜头按分镜生成(引用已锁资产,机位驱动渲染)
- **配音按台词行走**:每镜头的台词(对白)由对应角色的声线配音 → **对白回来了**;旁白是旁白、对白是对白,分开
- 装配(多轨对齐 + 交付门质检)

## 3. 对白 vs 旁白的显式区分(根治"只有旁白")

主线崩坏成"只有旁白",根因是它把剧本叙述文字整段丢给配音,没有"谁说话"的概念。修复:

- **②剧本级**:分场时区分"叙述"和"人物对白"
- **④分镜级**:台词行强制标注 `speaker`(哪个角色说),对白挂到该角色的声线;无 speaker 的是旁白,走旁白声线
- **⑤配音级**:多角色对话镜头,按台词行的 speaker 分别用各角色声线合成(接 SPEC-002 B1 的多声线能力)
- **验收**:一个多角色对话镜头,产出的是"角色A用A的声音说A的台词、角色B用B的声音说B的台词",不是一个旁白从头念到尾

## 4. readiness:五级锁定状态机(作品级)

```
作品状态:concept_draft → concept_locked → screenplay_draft → screenplay_locked
         → designlist_draft → designlist_locked → shotlist_draft → shotlist_locked
         → generating → done

每个 _locked 需要人审核通过(AI导演,人监制)。
未 locked,下游 _draft 禁止启动 —— 这是硬闸门,不是建议。
```

- 每一级 AI 生成 `_draft`,人审核修改后置 `_locked`。
- **允许回退**:下游发现问题可退回上游改(如分镜时发现人物设定不对,退回③改 DesignList),但退回上游 = 下游全部失效重来,强制保持一致性。
- 这把 v3.1 镜头级 readiness 升级为**作品级导演流水线 readiness**。

## 5. 出品形式如何调这条主线(tongjian / 短剧 / 单片)

**主线打造到能用后,所有出品形式复用这五级链,只在特定级注入自己的特化:**

| 出品形式 | 注入点 | 注入什么 | 下游 |
|---|---|---|---|
| **tongjian(资治通鉴)** | ①②之间 + ② | 文言原文 → 白话剧本转换(通鉴专属,留 tongjian);编年体史料的立意/剧本特化 | ③④⑤全用主线 |
| **短剧** | ①② | 网文/小说 → 立意 + 白话剧本(SPEC-001 的故事解析归入此) | ③④⑤全用主线 |
| **单片** | ① | 一句话意图 → 立意(最薄,②③④自动最小化) | ③④⑤全用主线 |

**关键:文言→白话转换是 tongjian 专属特化(只有通鉴用),留 tongjian;但它注入的位置是主线②剧本级,产出的白话剧本进主线的③④⑤。** 这符合 SPEC-002 判据:专属逻辑留特化前端,通用流水线用主线。

**这就是你要的"主线能用了,tongjian/短剧来调用"**:导演流水线(立意→剧本→设计→分镜→生成)是主线的,各出品形式只在立意/剧本级注入素材特化,下游全部继承。

## 6. 实施顺序(建议 CC)

```
阶段1(补齐缺失的导演层骨架,最小闭环):
  ③ 设计清单(AI从剧本分解场景/人物/衣着 → 锁定资产)—— 根治"乱跳",最优先
  ④ 分镜头剧本(切镜头 + 台词行带speaker)—— 根治"只有旁白"
  五级 readiness 状态机(上游锁定才开下游)
  验收门 G1:一段素材 → 走完①②③④ → 每镜头引用已锁资产 + 带对白台词 → ⑤生成
            人物/场景不跳 + 有对白(非纯旁白)

阶段2(立意与剧本的AI生成质量):
  ① 立意生成 + 审核锁定
  ② 白话详细剧本生成 + 审核锁定(含文言→白话,tongjian注入)
  验收门 G2:文言素材 → 白话详细剧本 → 下游产出的对白是白话

阶段3(出品形式接入):
  tongjian/短剧/单片 改为调用主线五级链,只注入各自立意/剧本特化
  退役 tongjian 自建的通用下游逻辑(SPEC-002 收敛)
  验收门 G3:tongjian 走主线链产出,不再崩坏,质量≥它自建版曾达到的水平
```

**G1 是重中之重**:③设计清单 + ④分镜带台词,是崩坏的两个直接根因,先把这两级补上,主线就从"能崩"变"能用"。

## 7. 明确不做

- **不让 AI 全自动跳过人审核**:AI 当导演不等于无人监制,每级 _locked 必须人拍板。全自动会在长作品上累积错误(实证:剧本不确定不能开下游,人审核是那道确定性闸门)。
- **不跳级生成**:任何绕过①②③④直接生成的旧路径,废弃。这正是崩坏的来源。
- **不把文言→白话做成通用能力**:它是 tongjian 专属,留特化,不进主线通用层。
- **不在③锁"占位"**:占位是④镜头级信息,③只锁"资产长什么样"。
- **不重造 L1 生成**:⑤复用现有生成/配音/装配,本 spec 只补①-④的导演层。
