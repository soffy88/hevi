# HEVI-SPEC-02: 电影级全自动流水线(通鉴 → 叙事短片)

- **状态**: DRAFT
- **版本**: v0.1
- **关系**: 与 HEVI-SPEC-01(解说级)共享 L0-L2(史料预处理/立意/剧本)与编排层基础设施;本 spec 只定义电影级分叉。**SPEC-01 已落地,不动。**
- **产出定义**: 3-8 分钟叙事短片。角色开口演戏(口型对齐的中文对白)、正反打镜头语言、跨镜头角色与空间一致、原生音效氛围。对标"动态漫剧/AI 短剧"上限,而非实拍电影
- **触发方式**: 导演台增加 `production_mode: commentary | cinematic` 开关,orchestrator 按 mode 路由到 SPEC-01 或本管线

## 2. C5 身份资产包(Character Identity Pack)——"角色向量化"的完整落地

每个有戏份的角色构建一个**一次制作、全片乃至跨卷复用**的身份包。这是电影级的地基,也是 294 卷量产时边际成本递减最陡的资产(智伯做一次,所有涉及智伯的章节复用)。

### 2.1 身份包内容与构建流程

```json
{
  "character_id": "C001",
  "name": "智伯",
  "identity_pack": {
    "canonical_portrait": "assets/id/c001/front.png",
    "multiview_grid": "assets/id/c001/grid9.png",
    "turnaround_video": "assets/id/c001/turn_5s.mp4",
    "expression_sheet": ["neutral", "倨傲", "暴怒", "惊恐"],
    "face_embedding": "assets/id/c001/arcface.npy",
    "voice": {
      "voice_ref_audio": "assets/id/c001/voice_8s.wav",
      "tts_voice_id": "cosyvoice:c001_cloned",
      "element_voice_binding": true
    },
    "costume_lock": "玄色深衣镶红边,束发玉冠",
    "gen_lock": {"kling_element_id": "...", "seedance_ref_slots": ["front", "left34", "right34"]}
  }
}
```

构建流程(全自动,每角色一次):
1. 文生图出正面权威像(同 SPEC-01 L5,VLM 审年代服制)
2. 权威像 → 图生多视角:九宫格(front/left34/right34/profile/back…)+ 表情表 —— 用图像模型的多视角一致生成,失败镜头用 img2img 补
3. 权威像 → 5 秒转身视频(图生视频,中性光照、素背景)—— 作为 Kling 视频元素参考(3-8s 角色视频)
4. TTS 生成 8 秒角色声线样本 → 声音克隆锚定,同一 voice 同时供:旁白外的 TTS 台词、以及生成端的元素声音绑定(视觉身份与声音在模型内绑死)
5. 提取人脸/形象 embedding(ArcFace 或 VLM embedding)入库 —— **这就是校验端的"角色向量"**:每个生成镜头抽帧算与身份包的余弦距离,低于阈值自动重roll

### 2.2 双端使用

- **生成端**: 每次视频生成调用注入身份包(Kling Elements 传 1-4 主体参考 / Seedance R2V 传多图+转身视频+声音样本 / Wan 2.7 九宫格)
- **校验端**: 门 CG5 对每个含角色镜头抽 3 帧 → embedding 距离 + VLM 服装锁定审("玄色深衣是否变白衣") → 不达标定点重roll
- **本地兜底**: 需要完全自主可控/离线时,Wan 2.2 + 每角色训练 LoRA(量产 294 卷时训练成本可摊薄),HunyuanCustom 做单主体镜头

## 3. 双时序架构(与 SPEC-01 的根本区别)

SPEC-01 是纯"音频先行":TTS 定一切时长。电影级必须反转一半:

| 素材类型 | 时序主导权 | 原因 |
|---|---|---|
| **旁白段**(narration/臣光曰) | 音频先行(同 SPEC-01):TTS 出时间戳 → 画面配长度 | 旁白压画面,画面服从声音 |
| **对白戏**(dialogue scene) | **视频先行**:音画由模型单次联合生成,实际时长以产出 clip 为准 | 口型、表演、音效在生成内已对齐,后期不可拆 |

因此 C8 剪辑层不再是"按时间戳拼接",而是 **EDL(剪辑决定表)驱动**:旁白段按 TTS 时长裁画面,对白戏按实拍时长入轨,幕结构由 L1 宪法约束总量,超时通过"旁白段画面留白压缩 + 对白戏 trim 尾帧"自动配平(允许成片时长 ±15% 浮动——电影级不追求秒级精确,追求节奏)。

对白台词的双保险:生成时在 prompt 内下台词(模型原生音频出口型),同时 ASR 反打生成 clip 的实际台词与剧本 diff —— 模型念错台词(漏字/改字)是电影级新增的失败模式,CG6 门必审;错词镜头重roll,3 次失败降级为"该句改旁白转述 + 角色画面不开口"。

## 4. C2.5 场景化改编 + C4 电影分镜

### 4.1 C2.5 场景化改编(剧本 → 场景剧本)

SPEC-01 的剧本是"解说词流",电影级需要先改编成**场景剧本(screenplay)**:

```json
{
  "scenes": [
    {
      "scene_id": "SC04",
      "slug": "内景·智伯府邸宴厅·夜",
      "characters": ["C001", "C004"],
      "space_anchor": "S003",
      "beats": [
        {"beat_id": "B01", "action": "智伯举杯起身,环视席间", "dialogue": null},
        {"beat_id": "B02", "action": "智伯讥讽段规", "dialogue": {"speaker": "C001", "text": "……", "quote_id": "Q007", "emotion": "狂傲"}},
        {"beat_id": "B03", "action": "段规垂目不语,指节收紧", "dialogue": null}
      ],
      "narration_bridge": {"position": "before", "line_id": "LN013"}
    }
  ]
}
```

红线继承:dialogue 仍只准改写自 chapter_ir.quotes;**action(动作/表演)是允许的合理虚构**,但门 CG2.5 审"动作不得改变史实因果"(段规可以"指节收紧",不能"拔剑")。这是电影级对史实红线的精确放宽:表演自由、情节锁死。

### 4.2 C4 电影分镜(场景 → 镜头组)

每个 scene 展开为镜头组,核心是三张"图":

1. **镜头语言规划**(LLM 按镜头语法生成):建立镜头(establishing)→ 对手戏正反打(shot/reverse-shot)→ 反应镜头(reaction)→ 特写强调。宪法 tone 决定节奏(肃杀 → 长镜头慢推;冲突 → 快切)
2. **空间连续性图**(spatial continuity graph):每个 scene 的 space_anchor 生成 1 张场景全景权威图(内容寻址,复用),该 scene 所有镜头的首帧从全景图裁切/重绘衍生,保证座次、烛台、门窗方位跨镜头一致;越轴检查用代码实现(180 度规则:同一对话轴两侧的机位不混用)
3. **生成打包规划**:相邻镜头按模型能力打包 —— 一场 4-6 镜头的对手戏打包为 Kling Multi Shot 单次生成(单 pass 内模型自保一致性);超长场景用首尾帧链式续镜(上一 clip 尾帧 = 下一 clip 首帧)

硬规则(来自实证 #4):
- `characters.length >= 2` 且景别 ∈ {close, medium_close} 的镜头 → **自动拆分为正反打两镜**
- 双人同框仅允许 {wide, full} 景别或过肩镜头(one clean face rule:一帧内只允许一张清晰主脸)

### 4.3 输出契约 `cine_shotlist.json`(节选)

```json
{
  "shots": [
    {
      "shot_id": "SH041",
      "scene_id": "SC04",
      "beat_ids": ["B02"],
      "pack_id": "PACK09",
      "shot_size": "medium_close",
      "camera": "slow push-in, 35mm, 烛光低调布光",
      "on_screen": ["C001"],
      "dialogue_inline": {"speaker": "C001", "text": "…", "voice_binding": "c001"},
      "first_frame_from": "SC04_anchor#crop_dais",
      "est_duration_s": 6,

```

## 5. C6 视频生成——多通道路由 + 两遍制

### 5.1 镜头路由表(orchestrator 按 shot 属性自动选通道)

| 镜头类型 | 首选通道 | 备选 | 理由 |
|---|---|---|---|
| 对白戏(开口) | Kling 3.0 Multi Shot(原生音频+元素声绑定+中文口型) | HappyHorse-1.0(fal)/ Seedance 2.0 | 口型+声音身份绑定是刚需 |
| 动作戏/复杂运动 | Seedance 2.0(R2V 多参考+联合音频) | Kling 3.0 | 参考容量大(9图3视频3音频),动作质量强 |
| 空镜/氛围/转场 | 本地 Wan 2.2(+场景 LoRA)或 Kling Fast/Draft | Seedance Fast | 无角色、成本敏感,本地跑省钱 |
| 旁白覆盖段画面 | 音频关闭档(Kling $0.112/s)或本地 | — | 声音走 TTS 轨,买原生音频浪费 |
| 高光镜头(dramatic_weight=5) | 更高档模型(Veo 3.1 / Seedance Pro) | — | 全片 2-3 个,预算倾斜 |

通道抽象为 oprim `video_generate(route, refs, prompt, audio_mode, first_frame, last_frame)`,路由表是 omodul 配置,模型换代只改配置不动管线。

### 5.2 两遍制(电影级迭代经济学的核心)

1. **Draft 遍**: 全片所有镜头用 Draft/Fast 档(5-20x 快、约 1/5 价)预演 → CG6 门全量审(身份 embedding、口型 ASR、穿帮 VLM、越轴代码审)+ LLM 导演审(节奏/表演意图)→ 产出"过审镜头清单 + 重拍指令"
2. **Final 遍**: 只对过审 prompt/参考组合出正片档;Draft 未过审的镜头带着修正指令重 Draft,直到过审或触发降级
3. 降级链: 对白镜头 3 次失败 → 转旁白转述+不开口表演 → 再失败 → 静帧+推拉(回落到 SPEC-01 手法,单镜头级别,不影响全片)

### 5.3 校验门 CG6(在 SPEC-01 G6 之上新增)

| 审项 | 方式 | 阈值动作 |
|---|---|---|
| 角色身份 | 抽帧 embedding 距离 vs 身份包 | 距离超限 → 重roll |
| 台词正确 | clip 音轨 ASR vs 剧本 diff | CER>3% → 重roll |
| 口型同步 | 音画同步检测(SyncNet 类打分) | 低分 → 重roll |
| 服装/道具锁定 | VLM vs costume_lock | 违锁 → 重roll |
| 穿帮/肢体崩坏 | VLM(六指、穿模、现代物件) | → 重roll |
| 越轴/空间跳变 | 代码(机位标记)+ VLM(背景一致) | → 换机位重拍 |

## 6. C8 声画精剪

- **EDL 组装**: 对白戏 clip 原声入轨(含模型生成的现场音效/氛围);旁白段 TTS 入轨压画面;交界处 J-cut/L-cut(音频先入/画面后出各 0.3-0.5s)—— 纯 ffmpeg 规则实现,立刻甩掉"PPT 感"
- **混音**: 旁白 -16 LUFS 主导;对白现场声 -18;BGM duck -24;全片响度归一
- **调色**: 宪法 palette 生成 3D LUT 全片统一套色(跨模型通道生成的镜头色温不一致,LUT 是最廉价的统一手段)
- **终审 CG8**: SPEC-01 G8 全项 + 跨镜头连续性抽查(VLM 看相邻镜头对:同场景光线方向/服装/道具是否跳变)

## 7. 成本与产能(3 分钟成片,约 35 个镜头 / 成片 190s)

| 项 | 估算 | 说明 |
|---|---|---|
| Draft 遍 | 190s × 2.2 roll × ~$0.03/s ≈ **$13** | Draft/Fast 档全片预演 |
| Final 对白戏(含音频) | ~70s × 1.3 roll × $0.168/s ≈ **$15** | 只重生成过审组合,roll 率低 |
| Final 其余镜头(无音频/本地) | ~120s × 1.3 × $0.06/s ≈ **$9** | 空镜走本地≈0 |
| 身份包(每新角色,一次性) | ~$1-2/角色,跨卷摊销 | 多视角+转身视频+声纹 |
| LLM(改编/分镜/各门)| ≈ **$3** | 场景剧本+导演审比 SPEC-01 重 |
| **合计** | **≈ $40-60/支**,全自动 2-4 小时 | vs SPEC-01 的 $1/30min;良品率目标:免人工干预直出率 ≥70%,其余触发降级仍能出片 |

可选人审位: `human_gate: off | draft_review | final_review` 三档开关。off = 全自动;draft_review = 只在 Draft 遍后人工替代 LLM 导演审(约 15 分钟/支,性价比最高的人力介入点)。

## 8. 实施路线

- **C-P0(2-3 周)**: 身份资产包管线 + 单场景对手戏打通(1 场戏 5 个镜头:建立镜头+正反打+反应),验证四件事:embedding 门有效性、中文口型良品率、Multi Shot 单 pass 一致性、正反打拆解规则。**验收: 智伯索地一场戏,全自动直出,身份不漂、台词不错、不越轴**
- **C-P1(3-4 周)**: C2.5 改编层 + 全片 EDL 剪辑 + 双时序配平 + 两遍制 → 整章成片
- **C-P2(持续)**: 路由表调优(各通道良品率/成本数据回流自动调权)、身份包库跨卷复用、LUT/镜头语法风格库、`human_gate` 数据对比(人审 vs LLM 导演审的良品率差)

## 9. 风险表(电影级新增)

| 风险 | 对策 |
|---|---|
| 模型念错中文台词 | ASR diff 门 + 台词短句化(每镜头 ≤2 句,实证最佳实践)+ 降级旁白转述 |
| 双角色同框身份混淆 | 分镜硬规则(one clean face)+ Multi Shot 单 pass 打包 |
| 跨通道视觉风格不统一 | 全片 LUT + 宪法 visual_style 注入所有通道 prompt + CG8 相邻镜头审 |
| API 成本失控 | 两遍制 + 路由表 + 每 run 预算熔断(超 $80 自动降级余下镜头) |
| 模型/API 快速换代 | 通道全部 oprim 抽象,路由表配置化;spec 不绑定任何单一模型 |

## 10. 渲染风格分支:动画片 vs 真人片

`production_mode: cinematic` 下新增第二维开关 `render_style: animated | photoreal`。两分支共享 C2.5/C4/C6/C8 骨架,差异集中在五处:

### 10.1 分支差异矩阵

| 维度 | animated(国风动画/动态漫) | photoreal(真人质感) |
|---|---|---|
| **对白时序** | 可保留音频先行(TTS 定时长):动画口帕宽容度高,或角色不开口走画外音+表演 | 必须视频先行:原生音画联合生成,口型零容忍 |
| **身份校验向量** | anime 专用 embedding 或 VLM 审;身份主要靠服装/发型/色彩符号,漂移容忍高 | ArcFace 人脸 embedding,量化最准,漂移容忍低 |
| **新增资产包** | **风格资产包**(style pack):风格参考图组 + 风格 LoRA + 负面清单,与身份包同级注入所有生成通道;风格漂移是本分支头号风险 | 无需风格包,但服化道考据审(甲胄/礼器/建筑)升级为独立 VLM 审项 |
| **物理/瑕疵门** | 宽松:风格化遮罩大半物理瑕疵,CG6 物理审降档 | 严格:手部/布料/皮肤全审,roll 率显著更高 |
| **合规新增** | — | 撞脸检测门(生成人脸 vs 公众人物库比对,命中即重roll);AI 生成标识在真实感内容上从严执行 |
| **通道路由倾向** | 动画强项模型(Vidu 系 / Wan+风格 LoRA 本地 / 即梦动画风) | Kling 3.0 / Seedance 2.0 / 高光 Veo 3.1 |
| **预估直出率 / 成本** | ≥85% / ≈$25-40/支(roll 率低+本地通道占比高) | ~60-70% / ≈$50-80/支 |

### 10.2 定位决策

- **animated 为主线**: 全自动率最高、成本最低、与通鉴题材审美先例(国风历史动画)匹配,294 卷量产走此分支
- **photoreal 为高端线**: 少量精品/品牌片,默认建议开 `human_gate: draft_review`
- 身份资产包按分支双版本构建(同一 character_id 下 animated_pack / photoreal_pack 并存),风格包仅 animated 分支持有
- 路由表、门阈值、降级链全部按 `(production_mode, render_style)` 二维键配置化,orchestrator 不新增逻辑

### 10.3 C-P0 修订

首战场景("智伯索地")以 **animated** 分支执行,新增第五个验证目标:风格包跨镜头锁定有效性(全场 5 镜头风格一致性 VLM 评分)。photoreal 分支的 C-P0 延后至 animated 全片跑通后启动。

## 11. 第二轮实证修订(2026-07)

### 11.1 参考工程纪律(写入 C5/C6 的 prompt 构造器)

1. **身份归参考,动作归 prompt**: shot prompt 中禁止出现身份描述(外貌/服装词),身份完全由参考资产承载——prompt 里的身份词会与参考图竞争导致漂移。C6 的 prompt 构造器加 lint:检测到 costume_lock 词汇出现在 shot prompt 中即报错
2. **参考集 2-4 张精选 > 6 张冗余**: 身份包注入时按镜头需求选 2-4 张(正面+三四分+全身/动态各一),不满注全部槽位
3. **参考集必含动态姿势**: 身份包构建流程(§2.1)第 2 步增补:九宫格外单独产出 1 张动作姿势参考,模型对角色运动方式的理解依赖动态参考
4. **台词短句化上限确认**: 每镜头 ≤2 短句(所有平台一致的最佳实践)

### 11.2 新增失败模式与对策

| 失败模式 | 实证 | 对策 |
|---|---|---|
| 快速运动破坏参考保真 | 高动态镜头 3/12 漂移 | C4 分镜规则:高动态镜头(战斗/奔逃)时长压至 ≤4s、拆多镜快切;CG6 对高动态镜头调低身份阈值容差、调高 roll 预算 |
| 漂移随时长累积(>8s 显著) | 多方一致 | 单镜头默认 ≤6s;长于 8s 强拆 |
| 复杂运镜在复杂场景 >4s 崩 | 动画分支实证 | 复杂场景只用简单运镜;大幅运镜留给空镜 |
| 稳定性伪信号 | 单次成功可能是 outlier | 身份包发布前过**稳定性预检**:同参考同 prompt 重生成 3 次,≥2 次过 CG5 门才允许入库(写入 SPEC-03 资产生命周期) |
