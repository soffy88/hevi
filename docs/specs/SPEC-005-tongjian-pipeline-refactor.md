# Hevi 通鉴管线改造 · SPEC-005

> 状态:草案 v0.1,可交 CC 实施
> 依赖:`SPEC-002`(主线是唯一能力权威源,tongjian 是特化出品形式)、`SPEC-003 v0.2`(五级导演链)、`SPEC-004`(SceneStage)
> 定调:**通鉴一集 = 讲解主干 + 演绎插段(形态 C)。两条生产路径,一个装配。**
> **关键:讲解段今天就能做(不需要角色/SceneStage/多人同框);演绎段走导演流水线,卡在多人同框——但单人演绎今天也能做。所以第一集选"商鞅立木"(单人为主),不等多人能力。**

---

## 0. 一集的结构

```
一集(5 分钟) = [讲解段] [演绎段] [讲解段] [演绎段] [讲解段] ...
                 ↑ ~3.5min             ↑ ~1.5min(3-4镜)
                 今天能做               走导演流水线

两条生产路径,最后一个装配。
```

**为什么是 C 不是 A/B:**
- 纯讲解(A)= 有声书配图,没有影视价值
- 纯演绎(B)= 卡在多人同框,且通鉴大量内容(背景/后果/评论)不适合演
- **C = 讲解承载信息密度,演绎承载情绪峰值。** 这也是纪录片的通行做法。

## 1. 四个通鉴专属前置(留 tongjian 自建)

按 SPEC-002 判据:只有《资治通鉴》场景用 → 留 tongjian,不进主线。

### 1.1 ★ 选段(chunking)—— 最关键的一步

**通鉴一卷动辄上万字、横跨数年。一卷 ≠ 一集。**

```
一卷原文
  → AI 切成若干「事件单元」(有起承转合的完整事件)
  → 每个事件单元 = 一集候选
  → 单元内标出「演绎点」
```

**演绎点判据(AI 提候选,人确认 —— Construction-First):**

| 特征 | 归为 |
|---|---|
| 有对白 / 冲突 / 决策瞬间 / 具体动作 | **演绎段** |
| 背景 / 铺垫 / 制度说明 / 后果 / 司马光评论 | **讲解段** |

**产出 `EventUnit`:**
```
EventUnit
├── source_ref              # 卷次 + 原文起止
├── title                   # "商鞅立木"
├── era / year              # 时间锚
├── summary                 # 一句话
├── segments[]              # 有序段落序列
│   └── Segment
│       ├── type            # narration(讲解) | drama(演绎)
│       ├── source_text     # 对应原文
│       ├── est_duration_s
│       └── order
└── characters[]            # 本单元涉及人物(演绎段建 Subject 用)
```

### 1.2 ★ 文言→白话:两套标准,别混用

| | 讲解段 | 演绎段 |
|---|---|---|
| 目标 | 让观众听懂这段历史 | 让人物说人话 |
| 手法 | **可意译、可展开、可补背景** | **口语化对白,不能文绉绉** |
| 例 | "秦孝公任用商鞅变法,但百姓不信任新法" | 商鞅:"谁能把这根木头搬到北门,赏十金。" |
| 注入点 | SPEC-003 ②剧本级 | SPEC-003 ②剧本级 |

**同一个 ②剧本级,两个不同的 prompt 模板。** 别用一套。

### 1.3 ★ 版权:只拉原文

```
✅ 通鉴原文(公版)
✅ 中华书局点校本的【原文部分】(点校本的原文仍是公版)
❌ 任何现代白话译本 / 注本的译文(有版权)
❌ 任何今人的解读文字
白话 = 我们自己做的(1.2)
```

**lint T1:** 拉取的文本若匹配已知译本特征(白话、现代标点密度、注释标记)→ 报错拒收。

### 1.4 史料画面来源(三档,按确定性排)

| 画面类型 | 来源 | 成本 |
|---|---|---|
| 地图 / 世系图 / 时间线 / 制度结构图 | **程序生成**(确定性,最准) | 零 |
| 古代场景 / 器物 / 服饰 | 资产供应链检索(v3.2 §5.0.1)或生成 | 低 |
| 人物 | **复用演绎段的 Subject canon**(一集内统一) | 零(已有) |

**关键:讲解段出现商鞅时,用的是演绎段那个商鞅的 canon。一集内人物形象统一。**

## 2. 讲解段生产路径(★ 今天就能跑通)

```
白话讲解稿
  → 分段(每段 = 一个画面单元,~10-20s)
  → 每段配画面(§1.4 三档)
  → TTS 旁白(平实叙述人设,见 §2.1)
  → 装配
```

**不需要:角色 Subject / SceneStage / 分镜 / 多人同框 / i2v**
**这条路径的所有依赖今天都已具备。**

### 2.1 旁白人设:平实叙述

```
voice_profile: narrator_plain
├── 语气:平实、清晰、不表演
├── 禁止:说书人腔、学者端架子、煽情
├── 语速:中等偏慢(信息密度高,给观众消化时间)
└── 理由:AI 配音演不出说书人的味,硬演必假;平实反而稳
```

**声线锁定(v3.2 §5.8):narrator 是一个独立声线资产,跨集恒定,且必须与任何剧中人声线明显区分。**

### 2.2 讲解段的画面节奏

```
一段讲解 ≈ 10-20s ≈ 1-2 个画面
禁止:一张图配 60 秒旁白(观众会走神)
禁止:每 2 秒换一张(信息过载)
```

**lint T2:** 单画面持续 >25s 或 <5s → 警告。

## 3. 演绎段走导演流水线(SPEC-003 五级链)

```
演绎点 → ①立意(继承本集基调,不重新立)
       → ②白话对白剧本(口语化)
       → ③设计清单(角色/场景/服装)
       → ③.5 SceneStage(走位/轴线/注意力)
       → ④分镜(带 speaker + target)
       → ⑤生成
```

**注入点:tongjian 只在 ①② 注入特化(文言原文 + 史实约束),③④⑤ 全用主线。** 完全符合 SPEC-002 §5 的表格。

### 3.1 ★ 降级:单人演绎先做

**通鉴里大量内容是单人镜头**——"王曰:善"、某人独自决策、某人宣读命令。**这些今天就能出**(单人身份已过关 + 朝向已落地)。

```
演绎段的镜头分级:
  L1 单人镜头           → 今天能做 ✅
  L2 双人对话(不接触)   → 等走位合成底图验证结果
  L3 多人/接触          → 等多人同框能力
```

**第一集选段原则:演绎点以 L1 为主。**

### 3.2 史实约束(通鉴专属,进 ①立意)

```
historical_constraints
├── era_visual        # 战国秦:服饰/建筑/器物的时代约束
├── no_anachronism    # 禁止年代错误(马镫、纸、椅子…)
├── source_fidelity   # 演绎不得违背原文事实
└── speculation_mark  # 原文未载的细节 = 合理推测,可加但需标注
```

**lint T3:** 演绎剧本中出现原文未载的**事件**(非细节)→ 警告需人确认。

## 4. 装配:讲解与演绎的接缝

### 4.1 接缝规则

```
讲解 → 演绎:
  旁白铺垫到位 → 一句引入("于是有了这样一幕")→ 切演绎
  ✗ 不要硬切(观众不知道为什么突然演起来了)

演绎 → 讲解:
  演绎在情绪/动作峰值后定格或收束 → 旁白接续
  ✗ 不要在演绎中途切走
```

### 4.2 声音分层(★ 关键)

```
narrator 声线   ≠ 任何剧中人声线    ← 硬约束
演绎段:对白(剧中人声线)+ 环境音,旁白静音
讲解段:旁白 + 轻环境床,无对白
```

**lint T4:** narrator 声线与任一 character 声线相同 → 报错。

### 4.3 视觉分层(可选但推荐)

```
讲解段与演绎段可用轻微视觉区分(如讲解段带轻微暗角/演绎段更饱和),
让观众潜意识知道"现在是在讲" vs "现在是在演"。
—— 但不要做成两种画风(那会显得是两个片子拼的)
```

## 5. 第一集垂直切片:商鞅立木

### 5.1 为什么选它

| 要求 | 商鞅立木 |
|---|---|
| 短 | 原文百余字,一集 5 分钟够 |
| 有名 | 观众有认知基础 |
| **演绎点以单人为主** | **商鞅立木、宣令、赏金 —— L1 镜头为主** |
| 有画面感 | 木头、城门、围观百姓、金子 |
| 有讲解价值 | 变法背景、立信的意义、司马光的评论 |

### 5.2 结构草案(5 分钟)

```
[讲解 ~60s] 秦孝公变法背景,商鞅其人,新法为何难行
[演绎 ~30s] 商鞅立木于南门,宣令赏十金        ← L1 单人为主 + 群众背景
[讲解 ~50s] 百姓为何不信(制度信任的历史语境)
[演绎 ~30s] 加赏五十金,有人搬木,当场予金      ← L1 为主
[讲解 ~90s] 立信之后新法得行;司马光论"信"
[演绎 ~20s] (可选)商鞅的一个决策瞬间特写      ← L1
[讲解 ~30s] 收束:这件事的后续与代价
```

### 5.3 验收门 G-T1

```
✓ 选段:一卷 → EventUnit 切分正确,演绎点标注合理(人确认)
✓ 白话:讲解稿听得懂;对白口语化不文绉绉
✓ 版权:全程无译本文字(lint T1 干净)
✓ 讲解段:画面节奏合规(T2 干净),旁白声线平实
✓ 演绎段:L1 镜头身份正确、朝向正确(复用 B5 已证能力)
✓ 装配:接缝自然,narrator 与剧中人声线明显区分(T4 干净)
✓ 史实:无年代错误(T3 干净)
✓ 成片:5 分钟完整,能看完不尴尬 ← 最终裁判是人眼
```

**G-T1 不过,不做第二集。**

## 6. 实施顺序

```
第一批(讲解段跑通 —— 今天就能做,不等任何能力):
  EventUnit schema + 选段(AI 提候选 + 人确认)
  文言→白话:讲解稿模板
  narrator 声线资产
  程序生成画面(地图/时间线)+ 检索/生成场景图
  讲解段装配
  验收:纯讲解版 5 分钟能出 ← 这本身已是可发布内容

第二批(演绎段接入 —— 用已有的单人能力):
  文言→白话:对白模板(与讲解稿模板分离)
  演绎点 → SPEC-003 五级链(①②注入 tongjian 特化)
  historical_constraints 进 ①立意
  L1 单人镜头生成(复用 B5 朝向能力)
  验收:一个演绎段能出

第三批(装配 + G-T1):
  接缝规则 + 声音分层 + 视觉分层
  lint T1-T4
  完整一集
  验收门 G-T1

第四批(等多人同框后):
  L2/L3 镜头解锁 → 双人对话/群戏演绎
  重做商鞅立木的群众围观镜头(第一集里降级处理的部分)
```

**注意第一批的独立价值:纯讲解版本身就是可发布内容。** 即便演绎段永远做不好,通鉴讲解视频也已经成立。**这是这条路线的安全垫。**

## 7. 明确不做

- **不拉任何译本**:只要原文,白话自己做
- **不把选段做成通用能力**:通鉴专属,留 tongjian(SPEC-002 判据)
- **不在第一集碰多人同框**:选段就按 L1 为主选
- **不做说书人腔**:AI 演不出来,硬演必假
- **不让讲解段与演绎段变成两种画风**:轻微区分即可
- **不为演绎而演绎**:适合讲的就讲,通鉴大量内容本来就不该演
